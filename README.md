## Overview

This repository contains the complete, reproducible implementation of the **Trustworthy Tabular AI Evaluation Framework (TTAF)** presented in the manuscript:

> *Threshold Transportability and Post-Training Threshold Adaptation in Clinical Tabular AI: A Multi-Dimensional Evaluation for Gestational Diabetes Prediction*  
> Submitted to *Artificial Intelligence in Medicine* (Elsevier)

The paper identifies **threshold transportability failure** — the inability of a decision threshold selected on internal data to generalise to new populations — as a critical but under-recognised failure mode in clinical AI. We propose a novel **Threshold-Instability Score (TS)** and demonstrate that post-training threshold adaptation can recover clinically usable recall (TabTransformer: 0.082 → 0.896 on PIMA) without retraining the underlying model.

---

## Key Contributions

1. **Primary:** Formal evaluation of threshold transportability as a distinct clinical AI reliability dimension
2. **Secondary:** Post-training threshold adaptation using the TS framework — a lightweight mitigation method under distribution shift

## Datasets

| Dataset | Role | N | Source |
|---|---|---|---|
| Internal (GestationalDiabeticDataSet.xlsx) | Model development and internal validation | ~3,525 | Institutional — available on request |
| PIMA (`diabetes.csv`) | External proxy cohort | 768 | [Kaggle / UCI](https://www.kaggle.com/datasets/uciml/pima-indians-diabetes-database) |
| Pregnancy (`Pregnancy_diabetes.csv`) | External proxy cohort | ~3,000 | [Kaggle](https://www.kaggle.com/datasets/rabieelkharoua/diabetes-health-indicators-dataset) |

> **Note:** PIMA and Pregnancy datasets were used as feature-aligned proxy cohorts for transportability assessment under distribution shift. They are not prospectively matched GDM validation populations. See manuscript Section 2.1 for the full methodological justification.

**Feature harmonisation:** All three datasets were mapped to a common five-feature predictor space: `age`, `pregnancies`, `bmi`, `diastolic_bp`, `glucose`.

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/YOUR-USERNAME/ttaf-gdm.git
cd ttaf-gdm
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Place datasets

Place the three dataset files in the repository root (see filenames above). The PIMA and Pregnancy datasets are publicly available from the links in the table above.

### 4. Run the notebook

```bash
jupyter notebook TTAF_Notebook.ipynb
```

Run all cells **in order from top to bottom**. All random seeds are fixed (`seed=42`). Total runtime is approximately 10–20 minutes depending on hardware.

## Threshold Framework

The **Threshold-Instability Score (TS)** is defined as:

```
TS(t) = w1·|ΔSpec| + w2·|ΔRecall| + w3·|ΔNet Benefit| + w4·|ΔCalibration Shift|
```

where each `|ΔM| = |M_internal − M_external(t)|` is the absolute difference between internal reference performance and target-cohort performance at threshold `t`.

**Weights used:** w1=0.20, w2=0.40, w3=0.30, w4=0.10 (recall weighted highest — missed GDM cases are the most clinically consequential error).

**Four strategies compared:**
1. Fixed transported threshold (baseline)
2. Quantile-matched threshold
3. Clinically constrained threshold (minimum recall 0.80)
4. TS-minimised threshold *(best performer)*

## Key Results

| Model | Dataset | Fixed recall | Adapted recall | Fixed specificity | Adapted specificity | TS reduction |
|---|---|---|---|---|---|---|
| Random Forest | PIMA | 0.567 | **0.985** | 0.682 | 0.178 | 26.6% |
| Random Forest | Pregnancy | 0.475 | **0.669** | 0.698 | 0.537 | 17.8% |
| TabTransformer | PIMA | 0.082 | **0.896** | 0.992 | 0.442 | 53.6% |
| TabTransformer | Pregnancy | 0.060 | **0.587** | 0.993 | 0.720 | 42.9% |

No model was retrained. All gains reflect post-hoc operating-point adjustment.
