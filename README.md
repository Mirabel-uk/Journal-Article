# Threshold Transportability and Post-Training Threshold Adaptation in Clinical Tabular AI

**A Multi-Dimensional Evaluation for Gestational Diabetes Prediction**

This repository contains the complete, reproducible analysis pipeline for the manuscript submitted to *Artificial Intelligence in Medicine* (Elsevier). It implements the **Trustworthy Tabular AI Evaluation Framework (TTAF)**, which evaluates threshold transportability and post-training threshold adaptation in clinical tabular AI models for gestational diabetes mellitus (GDM) prediction.

## Overview

Machine learning models for clinical prediction are commonly evaluated using discrimination metrics such as ROC-AUC. However, these metrics do not capture threshold-dependent decision behaviour under real-world deployment conditions. This study investigates **threshold transportability** — the ability of a decision threshold selected in one population to remain clinically usable in another — as a critical but under-recognised failure mode in clinical tabular AI.

Two models are compared on a harmonised five-feature predictor space (maternal age, number of pregnancies, BMI, diastolic blood pressure, plasma glucose):

- **Random Forest (RF)** — an ensemble baseline for structured clinical data
- **TabTransformer (TT)** — a deep tabular architecture using attention-based contextual embeddings

Both models are trained on an internal GDM cohort (n≈3,525) and evaluated on two external proxy cohorts — PIMA and Pregnancy — under feature-aligned distribution shift.

---

## Key Findings

1. **Both models achieve near-perfect internal discrimination** (RF ROC-AUC 0.998; TT ROC-AUC 0.995), confirmed as genuine — not an artefact of overfitting — via a structured leakage audit (train–test AUC gap <0.001; single-feature glucose AUC >0.95).

2. **TabTransformer recall collapses from 0.945 internally to 0.097 on PIMA** at its deployment-oriented threshold (TT = 0.55, selected on the calibration subset under a recall ≥ 0.95 constraint), despite retaining moderate external ROC-AUC (0.753). This demonstrates threshold transportability failure as a hidden failure mode invisible to discrimination metrics alone.

3. **Post-training threshold adaptation using the novel Threshold-Instability Score (TS)** recovers TT recall from 0.097 to 0.937 on PIMA without retraining — a 65.9% relative TS reduction — at the cost of a clinically expected specificity reduction.

## The Trustworthy Tabular AI Evaluation Framework (TTAF) — Notebook Structure

The notebook is organised into a setup phase and 14 main sections:

1. **Notebook setup** — reproducibility seed (42), output directory structure
2. **Harmonise the internal dataset** — maps raw variables to the five-feature schema (`age`, `pregnancies`, `bmi`, `dia_bp`, `glucose`)
3. **Load and harmonise external datasets** — PIMA and Pregnancy mapped to the same schema
4. **Preprocessing** — median imputation (RF); median imputation + standardisation (TT), fitted on internal training data only
5. **Train final models** — RF (300 trees, balanced class weights) and TabTransformer (4 attention heads, 2 layers, embedding dim 32); includes repeated-seed validation (10 seeds) and a high-discrimination leakage audit
6. **Final threshold optimisation** — threshold sweep (0.05–0.90) for sensitivity analysis
7. **Final performance table** — Table 3 and the full post-training threshold adaptation framework (TS score, four strategies, weight sensitivity)
8. **Calibration and recalibration** — Platt scaling, isotonic regression, ECE, Brier score
9. **Decision curve analysis** — net benefit across threshold ranges
10. **Main figures** — ROC curves across all model–dataset combinations
11. **Subgroup robustness analysis** — stratified by age, BMI category, and parity
12. **Statistical comparison: McNemar test** — paired comparison of RF and TT at deployment thresholds
13. **XAI stability** — SHAP stability, SHAP–LIME cross-method agreement
14. **Final export checklist** — confirms all outputs saved to `final_journal_outputs/`

**Supporting sections** (run after Section 5, before Section 7):
- **Calibration Protocol, Seed Control, Threshold Selection** (Cells 85–89) — standalone reproducibility utilities and calibration sanity checks for the supporting analyses
- **Explainability Reliability Evaluation** (Cells 90–92) — 50-instance Spearman rank correlation and perturbation robustness (σ=0.05), producing Table 8D
- **Unified Clinical Threshold Selection and Multi-Instance XAI Reliability** (Cells 95–97) — resolves the two-threshold methodology: selects deployment-oriented thresholds (RF=0.40, TT=0.55) on the **calibration subset** under a recall ≥ 0.95 constraint, as described in manuscript Section 2.5

## The Threshold-Instability Score (TS)

The TS score is a novel, clinically weighted composite criterion for post-training threshold adaptation:

```
TS(t) = w1·|ΔSpecificity| + w2·|ΔRecall| + w3·|ΔNetBenefit| + w4·|ΔCalibrationShift|
```

where each Δ is the absolute difference between internal reference performance and target-cohort performance at threshold *t*. Weights (w1=0.20, w2=0.40, w3=0.30, w4=0.10) prioritise recall, reflecting the clinical cost of missed GDM cases in a screening context. Four adaptation strategies are compared: fixed transported, quantile-matched, clinically constrained, and TS-minimised thresholds.

## Running the Notebook

1. Place the three data files (`GestationalDiabeticDataSet.xlsx`, `diabetes.csv`, `Pregnecy_complication_dataset.csv`) in the same directory as the notebook.
2. Install dependencies: `pip install -r requirements.txt`
3. Open the notebook in Jupyter and select a kernel with the required packages installed (the kernel must have PyTorch available for TabTransformer).
4. **Cell 3** is a data file diagnostic — run this first if you encounter a `NameError`. It lists your working directory contents and confirms the three data files are found before the pipeline proceeds.
5. Run **Kernel → Restart & Run All**. The reproducibility seed (42) ensures identical results on every run.
6. Run order matters: the **Unified Clinical Threshold Selection** cells (95–97) must run **before** Section 7 (final performance table), Section 11 (subgroups), and Section 12 (McNemar), as these depend on `RF_THRESHOLD_FINAL` and `TT_THRESHOLD_FINAL`. "Restart & Run All" handles this automatically since these cells appear earlier in execution order.
7. Outputs are saved to `thesis_outputs/`, `final_journal_outputs/`, and `paper_tables/`.
